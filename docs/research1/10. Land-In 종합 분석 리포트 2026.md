# Land-In 종합 분석 리포트: 시장 동향·기술 아키텍처·포인트 정책·수익성

> **작성일:** 2026년 5월 12일 | **버전:** v1.0 | **대상:** Land-In 프로젝트 팀 (파트너 설득용 사전 검토 메모)

***

## 1. Executive Summary

Land-In은 한국을 방문하는 외국인 관광객을 위해 단일 앱·단일 NFT 패스로 결제·보상·지역 혜택을 통합 제공하는 광역 관광 플랫폼이다. 기술 구조는 **B2B 정산(Hyperledger Fabric 프라이빗 블록체인) + B2C 보상(Polygon 퍼블릭 NFT) + 오프체인 개인정보 DB**의 하이브리드로 설계되어 있다.

**핵심 판단 5가지:**

1. **시장 타이밍 적합** — 2025년 방한 외국인 1,893만 명 역대 최고, 2026년 2,036만 명 돌파 전망이지만, 서울 방문 집중률 77.3%라는 구조적 분산 실패 문제가 존재하여 Land-In의 광역 통합 가치는 정책·시장 양쪽에서 요청된다.[^1][^2][^3]
2. **NFT 발행 비용 현실적으로 낮음** — Polygon PoS 기준 NFT 1건 민팅 비용 $0.01 미만, Gasless Relayer 적용 시 사용자 부담 0원 구현 가능하나 운영사가 릴레이어 비용 전액 부담해야 하는 구조 설계 필수.[^4][^5]
3. **Hyperledger Fabric 인프라 비용 스케일 가능** — PoC 단계 월 $50 이하, 소규모 프로덕션 월 $150~600, 엔터프라이즈 연 $60,000 수준의 비용 옵션 존재.[^6][^7]
4. **지역화폐 ↔ 플랫폼 포인트 전환은 법적 회색지대** — 지역사랑상품권법(2025.8.26 개정)은 외국인 사용을 명시 금지하지 않으나, 플랫폼 포인트와의 쌍방 전환은 선례·규정 모두 없음. API 제휴형 구조(플랫폼 포인트→지역화폐 결제 대리)로 우회 가능.[^8][^9][^10]
5. **항공사 마일리지 ↔ 플랫폼 포인트 전환은 항공사별 정책에 종속** — 현재 대한항공 네이버페이 포인트 전환 비율 25:1, 아시아나는 탑승 마일 1:1·제휴 마일 0.82:1 기준 대한항공으로 통합 진행 중. LCC는 마일리지 프로그램 미보유 또는 단순 포인트 구조로 협상 난이도가 FSC 대비 낮음.[^11][^12][^13]

## 2. 관련 시장 동향

### 2.1 방한 인바운드 관광 시장

2025년 상반기 방한 외국인 관광객은 882.6만 명으로 역대 최대치를 기록했으며, 이는 2024년 동기 대비 14.6% 증가한 수치다. 연간 기준으로는 2025년 1,893만 명(역대 최고), 2026년 2,036만 명 돌파가 전망된다. 국가별로는 중국(615만 명), 일본(384만 명), 대만(193만 명), 미국(166만 명) 순이며, 특히 미국 시장은 강달러 효과로 팬데믹 이전 대비 60% 이상 성장이 예상된다.[^14][^15][^16][^17][^1]

그러나 **질적 지표는 악화** 중이다. 2025년 상반기 1인당 평균 지출은 1,012달러로 2019년 동기(1,225달러) 대비 17.4% 감소했고, 관광수입도 89.4억 달러로 2019년 동기 대비 13.6% 줄었다. 방문자 수가 늘었지만 소비는 오히려 감소하는 구조적 문제가 발생하고 있으며, 2025년 1분기 관광수지 적자가 33억 달러에 달하는 현실은 Land-In의 "소비 증대·지역 분산" 가치를 강화하는 외부 논거가 된다.[^16][^18]

### 2.2 지역 편중 문제와 정책 기회

외국인 방문 비율은 서울(77.3~78.4%), 부산(16.4%), 제주(10.5%)로 집중 심화 추세다. 이에 정부는 2026년부터 농어촌 인구감소지역 여행 시 경비의 50%를 지역화폐로 환급(개인 10만 원, 단체 20만 원 한도)하는 정책을 시행 중이며, 문체부-KTO-코레일-클룩 협력 하에 외국인 전용 KTX 예매 서비스를 2026년 4월 개시하고 6,000원 할인권을 제공하는 등 외국인의 지방 접근성을 높이는 정책이 가시화되고 있다. Land-In의 광역 통합 플랫폼 구조는 이 정책 기조와 직결되며, 지자체의 외국인 관광 예산을 B2B SaaS 형태로 흡수할 수 있는 근거가 된다.[^19][^20][^21][^2][^3]

### 2.3 지역화폐 시장 현황

2026년 지역화폐 연간 발행 규모는 약 24조 원으로 예상된다. 제주 탐나는전의 경우 2026년 2월 누적 발행액 2조 4,485억 원, 앱 가입자 28만 명이며, 2025년 역대 최대 인센티브 예산 716억 원으로 7,300억 원 발행·6,800억 원 사용을 달성했다. 탐나는전 QR 플랫폼은 도입 이후 가맹점이 700개에서 19,651개(전체 40.4%)로 확대됐고 QR 결제액은 3,243% 증가했으며, 2026년 해외 결제 범위를 11개국 21개사에서 18개국 35개사로 확장 계획 중이다. 서울사랑상품권은 한글 성명 인증 구조로 사실상 외국인 접근이 제한되어 있었으나, 현재는 영문 성명 가입이 가능하도록 개선된 상태다.[^22][^23][^9]

### 2.4 NFT 관광 시장 동향

제주 NOWDA NFT 관광카드는 제주 NFC카드와 Npay NFT를 연계하는 국내 첫 사례로, 3.0 버전(2026년 1월 출시)에서 QR 결제와 NFT 발행을 통합했다. 탐나는전과 연계되어 관광 보조금을 지역화폐로 지급하는 구조를 채택했다. 비짓부산패스는 2023년 2월 출시 이후 2024년 기준 211개 가맹점·31개 판매처를 운영하며, 2024년 누적 판매 2,826건·누적 사용자 217만 명 수준이다.[^24][^25]

## 3. 기술 아키텍처 분석

### 3.1 하이브리드 블록체인 구조 설계

Land-In의 기술 스택은 4개 레이어로 구성된다.

| 레이어 | 기술 | 역할 | 온체인/오프체인 |
|--------|------|------|----------------|
| L4 | Land-In 앱 | Social Login, Gasless NFT, 다국어, QR 결제 | — |
| L3 | Polygon / BASE | 입국 NFT, 지역 스탬프, Explorer Pass | 온체인 |
| L2 | 오케스트레이션 API | 결제 라우팅, 지역화폐 변환, KYC | 오프체인 |
| L1 | Hyperledger Fabric | 다자 정산, KYC 해시, 탑승 검증 | 프라이빗 온체인 |
| L0 | 파트너 시스템 | 항공사 PSS, 지자체 지역화폐, POS | 오프체인 |

**온체인/오프체인 저장 원칙:**
- **Fabric 저장:** 다자 정산 트랜잭션, 승인 로그, KYC 해시(개인정보 원본 미저장), 탑승 검증 이벤트
- **Polygon NFT 저장:** 소유권 기록, 발행 이력, 멤버십 등급 증명
- **오프체인 DB/IPFS:** 상세 이미지, 가맹점 카탈로그, 개인정보 원본, NFT 메타데이터

### 3.2 Hyperledger Fabric (프라이빗 블록체인)

**Fabric을 선택하는 이유:** 서울시·부산시·강원도·제주도·KTO·항공사 등 서로 다른 법인들이 단일 중앙 DB를 신뢰하지 않는 구조에서, 신원이 확인된 기관 간 다자 정산을 프라이빗하게 처리하고 지역화폐별 회계 분리를 유지하는 데 Fabric의 채널(Channel)과 Private Data Collection 구조가 적합하다.

**Fabric 인프라 비용 구조:**

| 단계 | 옵션 | 비용 | 적합성 |
|------|------|------|--------|
| PoC/MVP | ChainLaunch Free + VPS (OVH/Hetzner) | $4~50/월 | Phase 1 PoC에 최적[^6] |
| 소규모 프로덕션 | VPS 직접 배포 (비Kubernetes) | $150~600/월 | Phase 2 초기[^7] |
| 중규모 프로덕션 | Kubernetes (AWS m5.xlarge) | $800~1,500/월 인프라 + 인건비[^6] | Phase 2 확장 |
| 엔터프라이즈 | ChainLaunch Pro | $60,000/년[^6] | Phase 3 / 규제산업 |

- Hyperledger Fabric 프레임워크 자체는 **오픈소스·무료 라이선스**로 기본 비용이 없다.[^26]
- 1,000 TPS 규모 4-노드 프로덕션 네트워크 기준, Hetzner 최적 구성 시 월 $600(연 약 $7,195)으로 구현 가능하다.[^7]
- Phase 1 PoC(속초·양양 파일럿 1,000명 규모)에서는 월 $50 이하로 완전한 Fabric 네트워크 운영이 가능하다.

**채널 구성 제안:**
- `항공사정산` 채널: 항공사-Land-In 간 탑승 검증 및 인센티브 정산
- `지역화폐정산` 채널: 지자체-Land-In-가맹점 3자 정산
- `감사로그` 채널: KTO·규제기관 접근 허용 감사 전용

### 3.3 Polygon 퍼블릭 NFT (보상 레이어)

**Polygon PoS 비용 구조:**

| 항목 | 수치 | 비고 |
|------|------|------|
| NFT 1건 민팅 비용 (일반) | $0.01 미만 | Polygon PoS 기준[^4] |
| 평균 가스비 | $0.002/트랜잭션 | Polygon 공식[^5] |
| POL 토큰 가격 | $0.10 (2026.5 기준) | MetaMask 기준[^27] |
| Gasless Relayer 사용 시 플랫폼 부담 | 건당 $0.002~0.01 | Biconomy/Gelato Relay 기준[^28][^29] |

**Gasless NFT 발행 구조 (사용자 가스비 0원 구현):**

Gasless Relayer(Biconomy 또는 Gelato Relay) 방식은 사용자가 서명만 하면 플랫폼 운영사가 가스비를 대납하는 구조다. 사용자 경험은 "NFT가 그냥 지갑에 생기는" 수준으로 단순화된다. 운영사는 월간 예상 NFT 발행량에 따라 가스비 예산을 선적립해야 하며, Polygon 혼잡 시 가스비가 일시적으로 10배 이상 급등하는 리스크가 존재한다.[^28][^29][^30]

**NFT 발행 비용 추정 (Phase 1 기준):**

```
Phase 1 목표: NFT 10,000개 발행
건당 Gasless 비용: $0.005 (Relayer + 가스비 합산 보수적 추정)
총 예상 비용: $50 (약 72,500원)
→ 사실상 무시할 수 있는 수준
```

**Ethereum 메인넷과의 비교:** Ethereum 메인넷의 NFT 단건 민팅 비용은 비수기 $8~15, 성수기 $50 이상으로 Polygon 대비 약 1,000배 이상 비싸다. 관광 보상용 대량 발행에는 Polygon 또는 BASE가 유일한 현실적 선택지다.[^4]

### 3.4 NFC — NTAG 424 DNA 기반 지역 방문 미션 인증

Land-In은 일반 제휴 가맹점 결제에는 QR 코드를 사용하되, **지역 방문 후 미션 달성 태그(스탬프 트리거)에는 NTAG 424 DNA NFC 칩**을 채택한다. 두 방식의 역할은 아래와 같이 명확히 분리된다.

**결제 vs. 방문 인증 역할 분리:**

| 기능 | 방식 | 이유 |
|------|------|------|
| 제휴 가맹점 QR 결제 | QR 코드 | 기존 POS 단말 호환, 도입 비용 최소화 |
| 지역 방문 미션 태그 | NTAG 424 DNA NFC | 위치 위변조 방지, 복제 불가, 앱 없이도 인증 가능 |

**NTAG 424 DNA의 핵심 특성:**

NTAG 424 DNA는 NXP가 설계한 보안 NFC 칩으로, NFC Forum Type 4 / ISO/IEC 14443-A 규격을 따른다. 핵심 보안 메커니즘은 **SUN(Secure Unique NFC) 메시지 인증**으로, 태그를 스캔할 때마다 AES-128 키와 내부 카운터를 조합하여 매번 다른 암호화 URL을 동적으로 생성한다. 이 구조 덕분에:[^31][^32][^33][^34][^35]

- **복제 불가:** 동일한 정적 URL을 복사해도 다음 스캔에서 서버 검증 실패 → 위치 위변조 원천 차단[^36][^37]
- **재사용 불가:** 내부 카운터가 매 스캔 증가하므로 동일 인증 코드 재사용 불가[^36]
- **앱 설치 없이 인증 가능 (Dynamic URL 방식):** iPhone 7+, 안드로이드 NFC 지원 기기에서 앱 없이도 휴대폰을 태그에 태그하면 브라우저가 자동으로 서버에 인증 요청 전송[^35][^31]
- **상호 인증 (Mutual Authentication, 앱 설치 시):** Land-In 앱 설치 사용자는 앱 내에서 챌린지-리스폰스 방식으로 더 강력한 상호 인증 수행 가능 — 스캔 정확한 타임스탬프까지 서버에 기록[^35]

**Land-In에서의 NTAG 424 DNA 동작 흐름:**

```
[사용자, 지역 방문 포인트(랜드마크·가맹 관광지) 도착]
    ↓
[스마트폰 NFC로 NTAG 424 DNA 태그 태깅]
    ↓
[태그가 SUN 인증 포함 동적 URL 자동 생성]
    ↓
[Land-In 앱 또는 브라우저 → 서버에 AES-128 검증 요청]
    ↓
[서버: UID + 카운터 + SUN 코드 검증 → 위치·시간 기록]
    ↓
[검증 성공] → Polygon Gasless NFT 스탬프 자동 발행 + L-Point 지급
[검증 실패] → 오류 처리 (복제/재사용 시도 차단)
```

**두 인증 모드 동시 지원 구조:**

NTAG 424 DNA는 Dynamic URL 방식과 Mutual Authentication 방식을 동시에 지원하도록 설정 가능하다. Land-In은 아래와 같이 단계적으로 운영한다:[^35]

- **앱 미설치 사용자 (Dynamic URL 방식):** 태그 → 브라우저 자동 열림 → 앱 설치 유도 랜딩페이지로 연결, 스탬프 발행 전 앱 설치 완료 유도
- **앱 설치 사용자 (Mutual Authentication):** 앱 내 NFC 스캔 → 상호 인증 → 정확한 방문 시각 기록 → NFT 스탬프 즉시 발행

**태그 단가 및 배포 비용 추정:**

| 항목 | 수치 | 비고 |
|------|------|------|
| NTAG 424 DNA 스티커 단가 | $0.3~0.8/개 (대량 주문 기준) | 인코딩 포함[^38][^34] |
| Phase 1 (속초·양양 30개 거점 기준) | 30개 × $0.5 = $15 | 초기 배포 비용 극소 |
| Phase 2 (4개 광역 500개 거점) | 500개 × $0.5 = $250 | 확장 시에도 비용 무시 가능 |
| 서버 SUN 검증 API 비용 | 거의 0원 | 자체 AES-128 검증 로직 구현 가능 |

**결제와의 구분 명확화:**

일반 결제 단말(POS)에 NFC를 강제하면 가맹점 교체 비용이 수십만~수백만 원 발생하나, 미션 태그는 스티커 형태 NTAG 424 DNA를 랜드마크·관광 거점에 부착하는 방식이므로 가맹점 기존 인프라에 영향이 전혀 없다. 이 구조는 결제 QR과 방문 인증 NFC를 완전히 독립시켜 각각의 도입 장벽을 최소화하는 핵심 설계 결정이다.

### 3.5 오프체인 개인정보 DB 구조

개인정보보호법(PIPA) 및 GDPR 규제 적용 시, 여권번호·KYC 정보·탑승 기록 등 개인 식별 정보는 Fabric 블록체인에 직접 저장하지 않는다. 대신 개인정보는 오프체인 암호화 DB에 저장하고, 그 해시값만 Fabric Private Data Collection에 기록하는 **해시 앵커링** 방식을 채택한다. 이는 KYC 검증의 무결성은 블록체인으로 증명하면서 개인정보는 PIPA·GDPR 준수 DB에서 별도 관리하는 구조다.

**오프체인 DB 저장 데이터 분류:**

| 데이터 유형 | 저장 위치 | 비고 |
|-----------|---------|------|
| 여권 OCR 정보 | 암호화 DB (AES-256) | 해시만 Fabric 저장 |
| KYC 결과 | 암호화 DB | 3rd party eKYC 결과 포함 |
| 탑승 기록 (PNR) | 임시 DB → 해시 Fabric | 원본 보유 최소화 |
| 가맹점 카탈로그 | 일반 DB (읽기 최적화) | 블록체인 불필요 |
| NFT 메타데이터 이미지 | IPFS + CDN | Polygon NFT의 tokenURI 참조 |
| 결제 상세 로그 | 암호화 DB | 5년 보관 의무 |

## 4. 포인트 발행 정책 설계

### 4.1 포인트 경제 (Tokenomics) 기본 구조

Land-In의 포인트(이하 "L-Point")는 법정통화와 1:1 연동된 **결제 크레딧 구조**로 설계한다. 투자 유인이 아닌 순수 관광 소비 인센티브 목적임을 명확히 해야 전자금융업 등록 및 가상자산 규제 충돌을 최소화할 수 있다.

**포인트 발행 재원 구조:**

| 재원 출처 | 발행 규모 기준 | 포인트 형태 | 비고 |
|-----------|-------------|-----------|------|
| 지자체 관광 보조금 | 지자체 투자액의 100% | 지역 제한 L-Point | 특정 지역 가맹점만 사용 가능 |
| 항공사 파트너십 fee | 계약 단가 × 탑승 수 | 광역 L-Point | 전 지역 사용 가능 |
| 플랫폼 운영사 자체 재원 | MAU 프로모션 예산 | 광역 L-Point | 앱 설치·첫 결제 인센티브 |
| QR 결제 수수료 일부 | 결제액의 0.3~0.5% | 적립형 L-Point | 결제 시 자동 적립 |

### 4.2 지자체 투자 구조 및 포인트 발행 메커니즘

**투자 방식:** 지자체는 외국인 관광 예산을 Land-In 운영사에 SaaS 이용료(플랫폼 이용 + 관광 인센티브 예산) 형태로 집행한다. 현금 직접 투자가 아닌 **관광 캠페인 예산 집행** 구조이므로 지자체 감사·회계 처리가 용이하다.

**포인트 발행 기준 (예시: 강원도 Phase 1):**

```
강원도 외국인 관광 예산 배정액: 5억 원 (Phase 1 파일럿)
├─ 플랫폼 이용료 (SaaS): 1억 원
├─ NFT 발행·운영 비용: 0.1억 원
└─ 관광 인센티브 L-Point 재원: 3.9억 원

L-Point 발행 규모: 3억 9,000만 포인트 (1포인트 = 1원 기준)
1인당 최대 발행 한도: 39,000 포인트 (약 27달러 상당)
Phase 1 목표 사용자 1,000명 기준: 1인당 평균 39,000 포인트 발행 가능
```

**포인트 지급 트리거 및 금액 기준:**

| 이벤트 | L-Point 지급액 | 조건 |
|--------|--------------|------|
| 입국 NFT 발행 (항공사 파트너 탑승) | 5,000~10,000 포인트 | 파트너 항공사 탑승 확인 후 |
| 지역 첫 방문 스탬프 | 2,000~3,000 포인트/지역 | GPS·QR 검증 |
| 가맹점 첫 결제 | 결제액의 5~10% | 지역 가맹점 한정 |
| 스탬프 N개 수집 시 등급 업그레이드 | 10,000~50,000 포인트 | 희귀 NFT 교환권 포함 |
| SNS 공유 | 500~1,000 포인트 | 1인 월 1회 한도 |

**포인트 유효기간:** 발행일로부터 1년 (미사용 만료 시 운영사 수익으로 귀속)

### 4.3 항공사 마일리지 ↔ L-Point 전환 정책

**현행 항공사 마일리지 정책 현황 (2026년 5월 기준):**

- **대한항공 스카이패스:** 네이버페이 포인트 → 마일리지 전환 비율 25:1 (2026년 2월 기준). 아시아나 합병 이후 탑승 마일 1:1, 제휴 마일 0.82:1로 통합 진행 중.[^12][^13]
- **아시아나항공:** 탑승 기반 마일리지 → 결제 금액 기반(운임 × 등급별 적립률) 방식으로 2025년 전환. 10년 유효기간 신설.[^39]
- **LCC (티웨이·파라타에어 등):** 별도 마일리지 프로그램 없거나 단순 포인트 구조 → 협상 여지 높음.

**Land-In의 항공사 마일리지 ↔ L-Point 전환 정책 설계:**

**Phase 1 (현실적):** LCC와의 단순 파트너십 쿠폰코드 방식
- 파트너 항공사(파라타에어, 비엣젯 등) 탑승권 QR/바코드 스캔(IATA BCBP) 또는 PNR 입력
- 검증 완료 시 입국 L-Point 자동 지급 (마일리지 전환 아닌 별도 지급)
- LCC 마일리지가 없으므로 "마일리지 전환" 개념 불필요

**Phase 2 (확장):** FSC 마일리지 ↔ L-Point 부분 전환
- 대한항공·아시아나와 MOU 협상 필요
- 전환 비율 협상 기준: 마일리지 1마일 ≈ 원화 15~25원 상당
- 실현 가능 구조: 마일리지 → L-Point 단방향 전환 (L-Point → 마일리지 역전환은 항공사 동의 필요)
- 전환 최소 단위: 100마일 이상, 전환 수수료 5~10% 적용 가능
- **주의:** 마일리지의 현금화 유도 해석 방지를 위해 L-Point 사용처를 관광 소비로 제한

**전환 불가 케이스 (명시적 제외):**
- L-Point → 현금 또는 현금성 자산으로의 전환
- L-Point → 타 항공사 마일리지 전환
- 미사용 L-Point의 현금 환급

### 4.4 지역화폐 ↔ L-Point 전환 정책

**법적 검토 결과:**

지역사랑상품권법(2020년 제정, 2025년 8월 개정)은 지역화폐의 외국인 사용을 명시 금지하지 않는다. 서울사랑상품권의 경우 2025년 중 영문 성명 가입을 허용하는 방향으로 개선됐다. 탐나는전(제주)은 2025년 3월부터 QR 외국인 결제를 허용하고 해외 결제 범위를 확대 중이다.[^9][^10][^22][^8]

**가능한 구조 (현행법 기준):**

| 전환 방향 | 가능 여부 | 구조 | 비고 |
|-----------|---------|------|------|
| L-Point → 지역화폐 결제 대리 | ○ 가능 | 플랫폼이 지역화폐로 대신 결제 | API 제휴 필요 |
| 지역화폐 → L-Point | △ 회색지대 | 지자체 MOU 필요 | 지역화폐 가치 이전 해석 우려 |
| 지역화폐 잔액 조회·연동 | ○ 가능 | 읽기 전용 API | 개인정보 동의 필요 |

**추천 구조 (Phase 1):**
L-Point를 직접 지역화폐로 전환하는 것이 아니라, **L-Point로 지역화폐 가맹점에서 결제하면 플랫폼이 해당 금액을 지역화폐 API를 통해 가맹점에 정산하는 "결제 대리" 구조**가 가장 법적 위험이 낮다. 사용자 입장에서는 "L-Point로 결제한다"고 느끼지만, 백엔드에서는 지역화폐 결제가 이루어진다.

**Hyperledger Fabric 정산 채널 설계:**
```
[사용자 L-Point 사용] → [Land-In 오케스트레이션 API] 
→ [Fabric 지역화폐정산 채널] 
→ [지자체 지역화폐 API] → [가맹점 POS 정산]
→ [정산 완료 이벤트 → Polygon NFT 스탬프 발행]
```

### 5.2 NFT 관련 수익 모델

**1. 2차 거래 로열티:** 희귀 스탬프 NFT·등급 업그레이드 NFT를 OpenSea·Blur 등 NFT 마켓플레이스에서 거래 시 스마트컨트랙트 설정 로열티 5~10% 수취. NFT가 희소하고 바이럴 될수록 수익 증가. 단, 2023년 이후 OpenSea의 로열티 강제 폐지로 인해 독자 마켓플레이스 구축 또는 로열티 강제 가능한 체인 선택 필요.
**2. 프리미엄 NFT 패스 직접 판매:** "Explorer Pass"(전 지역 프리미엄 혜택) NFT를 10달러~30달러에 판매하는 B2C 수익 모델.
**3. NFT 광고/스폰서십:** 기업·브랜드가 스폰서하는 Limited Edition NFT 발행 수수료.
**4. NFT 데이터 기반 B2B:** 방문 패턴 데이터(어익명화)를 지자체·항공사에 제공하는 B2B 데이터 수익.

### 7.2 전자금융업 관련 핵심 쟁점

L-Point가 선불전자지급수단으로 해석되면 전자금융업 등록이 필요하다. 회피 조건:
- 발행 한도: 1인당 50만 원 이내 (소액 선불)
- 사용처 제한: 제휴 가맹점만 사용 가능
- 현금화 불가: 환금성 없음
- 발행 목적: 관광 인센티브 (영리 목적 아닌 정책 목적 명시)

위 조건을 모두 충족할 경우 소액 선불 특례로 등록 면제 가능하나, 법무법인 검토를 통한 의견서 확보 권장.

- **수집·이용:** 서비스 이용계약 이행 (관광 혜택 제공) 목적
- **보유 기간:** 목적 달성 후 즉시 파기 원칙 (여권 정보 KYC 완료 후 즉시 파기)
- **국외 이전:** 파트너 항공사(외국계) 데이터 공유 시 표준 계약 조항(SCC) 또는 동의 필요
- **Fabric 저장 원칙:** 개인 식별 정보 원본 Fabric 저장 금지 — 해시값만 저장

## 9. 단계별 실행 전략

### Phase 1: PoC (0~6개월, 속초·양양)

**목표:** 외국인 1,000명, QR 결제 5,000건, NFT 10,000개, 지역화폐 연동 1개(강원)

**추천 파트너:**
- 항공사: 파라타에어 또는 비엣젯 (LCC, 마일리지 없음 → BCBP 스캔 방식)
- 지자체: 강원도 관광재단 (외국인 방문 +124% 성장 중)
- 결제 백본: Trip.PASS (기존 외국인 eKYC·PG 인프라 활용)
- Fabric 인프라: ChainLaunch Free + VPS ($50/월 이내)

**연동 방식 (현실적):**
- 탑승 검증: IATA BCBP 스캔 (탑승권 QR 스캔) → 수동 PNR 입력 폴백
- 지역화폐 연동: 강원도 지역화폐 API 읽기 전용 연동 + 결제 대리 구조
- NFT: Polygon Gasless (Biconomy) — 사용자 가스비 0원

**절대 과장 금지:**
- "실시간 항공사 PSS 연동" (Phase 1 불가)
- "모든 지역화폐 직접 전환" (법적 미검증)
- "FSC 마일리지 전환" (협상 완료 전 불가)

### Phase 2: 확장 (6~18개월, 4개 광역)

**목표:** MAU 10,000명, 4개 광역 연결, 항공사 2~3개, 지자체 MOU 확대

**추가 연동:**
- 항공사 webhook 연동 (제한적 이벤트 공유)
- OTA (Klook) API 연동으로 배포 채널 확대
- Hyperledger Fabric 채널 확장 (부산·제주 지역화폐 채널 추가)
- Trip.PASS의 DID 기반 KYC 공유 구조 협의

### Phase 3: 글로벌 (18개월+)

**목표:** FSC 연동, OTA 글로벌 배포, NFT 마켓플레이스, 글로벌 인지도

**추가 구조:**
- 대한항공/아시아나 PSS NDC API 연동 (DPA 계약 완료 후)
- 마일리지 ↔ L-Point 공식 교환 서비스 출시
- Agoda/Klook 화이트레이블 NFT 패스 공급

### Land-In의 해외 비교 우위

- **Fabric + Polygon 하이브리드:** B2B 다자 정산(Fabric)과 B2C 바이럴 보상(Polygon NFT)을 동시 해결하는 구조는 기존 사례에서 찾기 어렵다.
- **정책 기조 부합:** 2026년 정부의 외국인 지방 분산 정책·지역화폐 고도화 방향과 정확히 일치한다.[^20][^21]

> **"Land-In은 지자체가 따로 쓰던 외국인 관광 예산을 하나의 측정 가능한 디지털 인프라로 통합하고, 항공사 탑승부터 지역 소비까지 데이터를 연결해 각 파트너의 KPI를 처음으로 수치화할 수 있게 해주는 유일한 플랫폼이다."**

### 실제 구현 관점에서 가장 먼저 검증해야 할 것

1. **Trip.PASS API 연동 가능 여부 확인** — 결제 백본의 기술·계약 조건이 Phase 1 실현 가능성을 좌우
2. **강원도 관광재단과의 지자체 SaaS 예산 배정 협의** — 재원 없이는 포인트 발행 불가
3. **Gasless NFT Relayer 실제 UX 테스트** — 이론상 $0이지만 사용자 경험상의 지연·오류 검증 필요
4. **L-Point의 선불업 등록 요건 법무 검토** — 발행 한도·사용처 설계 확정 전 법적 의견서 필수
