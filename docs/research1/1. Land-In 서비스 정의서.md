# Land-In 서비스 정의서

> **문서 목적:** 이 정의서는 Perplexity가 후속 조사를 수행할 때 "우리 서비스가 무엇인지"를 빠르게 판단하기 위한 단일 참조 기준 문서다. 기술 상세·시장 조사·투자자용 수치는 이 문서가 아닌 별도 보고서에서 다룬다.

***

## 1. 한 문장 정의

**Land-In은 한국을 방문하는 외국인 관광객이 서울·부산·강원도·제주 등 다양한 한국의 지역들을 방문할 때 단 하나의 앱과 NFT 패스로 이동·결제·보상받을 수 있게 하는 광역 통합 관광 플랫폼이다.**

***

## 2. 해결하는 문제

### 외국인 관광객 관점

현재 외국인이 한국 여러 광역을 여행하려면 아래의 파편화된 시스템을 각각 설치·인증해야 한다.[^1]

| 지역 | 현행 시스템 | 외국인 한계 |
|------|-----------|-----------|
| 서울 | 서울사랑상품권 | 자치구별 별도, 외국인 접근 어려움 |
| 부산 | 부산페이(동백전) | 부산 가맹점만 사용 가능 |
| 강원도 | 강원사랑상품권 | 외국인 전용 통합 패스 없음 |
| 제주 | 탐나는전 + NFT 관광카드 | 제주 단독, 광역 연계 불가 |

결과적으로 외국인은 멀티시티 동선을 이동할 때마다 새 앱을 설치하고, 현금·카드로 반복 결제하며, 지역별 혜택을 놓친다. **플랫폼이 없는 게 아니라, 플랫폼이 너무 많아서 생기는 문제다.**[^1]

### B2B(지자체·항공사·KTO) 관점

각 지자체와 항공사는 외국인 유치 효과를 측정하고 서로 정산하는 공통 인프라가 없다. 결과적으로 외국인 지출의 66%가 서울에 집중되고, 다양한 지방 지역으로의 분산이 정책 목표임에도 인센티브 구조가 없어 실행되지 않는다.[^1]

***

## 3. 우리 서비스가 하는 것 (핵심 기능 3가지)

### ① 단일 앱·단일 패스로 4개 광역 이용

외국인은 입국 시 Korea Multi-Pass 앱 하나를 설치하고 신용카드로 충전한다. 서울·부산·강원·제주 어디서든 QR 하나로 결제된다. 내부적으로 Hyperledger Fabric이 결제 위치에 따라 자동으로 해당 지역화폐로 변환·정산한다.[^1]

> 사용자 경험: "앱 하나, 지갑 하나, 어느 도시든 그냥 쓴다."

### ② 방문할수록 쌓이는 NFT 스탬프

도시를 방문하거나 특정 가맹점에서 결제하면 Polygon 기반 NFT 스탬프가 자동 발행된다. 스탬프는 소유 가능하고 SNS에 공유 가능하며, 일정 수 수집 시 등급 업그레이드 NFT로 교환된다. 가스비는 없다(Gasless 릴레이어 방식).[^2][^1]

> 사용자 경험: "도장 모으듯 도시를 모은다."

### ③ 항공사 탑승 → 자동 혜택 연동

파트너 항공사로 입국하면 탑승 기록이 Hyperledger Fabric Private Data Collection에서 검증되고, 즉시 입국 NFT + 지역화폐 크레딧이 지급된다. 항공사에게는 탑승 → 지역 소비 데이터가 연결된 로열티 효과가 생긴다.[^3][^1]

> 사용자 경험: "비행기 내리자마자 혜택이 도착해 있다."

## 4. 왜 일반 앱이나 DB로는 안 되는가

| 질문 | 이유 |
|------|------|
| 왜 중앙 서버로 안 되나? | 서울시·부산시·강원도·제주도·KTO·항공사는 서로 다른 법인이다. 어느 한 기관이 단일 중앙 DB를 운영하면 타 기관이 신뢰하지 않는다. |
| 왜 Hyperledger Fabric인가? | 신원이 확인된 기관들 간 다자 정산을 프라이빗하게 처리할 수 있다. 지역화폐별 회계 분리를 유지하면서도 단일 사용자 경험을 제공하는 유일한 구조다.[^3][^1] |
| 왜 Public NFT(Polygon)인가? | 사용자 소유권, 2차 유통, 소셜 공유가 필요하다. Fabric만으로는 사용자-facing 바이럴 효과를 만들 수 없다. Lufthansa UpTrip, Singapore Airlines KrisPay 모두 사용자 레이어엔 퍼블릭 체인을 활용했다.[^2][^1] |
| 왜 하이브리드인가? | B2B 정산(Fabric) + B2C 보상(NFT)을 동시에 풀어야 하는 문제다. 각각 최적의 도구가 다르다. |

## 5. 이해관계자 구조

```
[외국인 관광객]
    ↓ Land-In 앱 사용
[Land-In 플랫폼]
    ↔ Hyperledger Fabric (정산 레이어)
    ↔ Polygon NFT (보상 레이어)
    ↕                    ↕
[지자체 (복수)]      [파트너 항공사]
서울/부산/강원/제주    LCC 1~2개 (Phase 1)
등 다양한 지역         탑승 데이터 제공
지역화폐 정산
    ↕
[KTO / Trip.PASS 백본]
교통 연계 (KTX·지하철·버스)
```

**각 주체의 역할**

- **외국인 관광객:** 앱 설치 → 충전 → QR 결제 → NFT 수집
- **지자체 (복수 지역):** 지역화폐 가맹점 네트워크 제공, Fabric 정산 채널 참여, 홍보 보조금 지원
- **항공사 (파트너):** 탑승 데이터 제공, 공동 마케팅, NFT 로열티 연계
- **KTO / Trip.PASS:** 교통(KTX·지하철·버스·택시) 인프라 백본 제공
- **운영사 (우리):** 플랫폼 개발·운영, API 오케스트레이션, NFT 발행, 수익 배분 정산

## 6. 수익 구조 (간략)

| 수익원 | 설명 |
|--------|------|
| 결제 수수료 | QR 결제 건당 1.5~2% (지역화폐 정산 수수료 포함) |
| 항공사 파트너십 fee | 탑승 데이터 연계 + NFT 로열티 발행 서비스 B2B 계약 |
| 지자체 플랫폼 이용료 | 각 지자체의 외국인 관광 예산 활용, SaaS 형태 |
| NFT 2차 거래 수수료 | 희귀 스탬프 NFT 마켓플레이스 수수료 (로열티 5~10%) |

## 7. 기술 구조 한눈에 보기

```
L4  사용자 앱 (Land-In)
     Social Login · Gasless NFT · 다국어 · QR 결제

L3  Public NFT 레이어 (Polygon / BASE)
     입국 NFT · 지역 스탬프 · Explorer Pass

L2  오케스트레이션 API (Trip.PASS 백본 확장)
     교통 연계 · 결제 라우팅 · 지역화폐 변환 · KYC

L1  Hyperledger Fabric (정산·감사 레이어)
     채널: 항공사정산 / 지역화폐정산 / 감사로그
     Private Data: 탑승기록 · KYC 해시

L0  파트너 시스템 (오프체인)
     항공사 PSS · 지자체 지역화폐 · 가맹점 POS · KTO
```

**온체인 / 오프체인 분리 원칙**
- Fabric에 저장: 다자 정산 트랜잭션, 승인 로그, KYC 해시, 탑승 검증
- Polygon NFT에 저장: 소유권, 발행 기록, 멤버십 등급 증명
- 오프체인(DB/IPFS): 상세 이미지, 메타데이터, 가맹점 카탈로그[^1]

## 8. 단계별 실행 계획

| 단계 | 기간 | 핵심 목표 | 파트너 구조 |
|------|------|----------|-----------|
| Phase 1 (MVP) | 0~6개월 | 속초·양양 강원 파일럿, 외국인 1,000명 | 항공사 1개 (파라타항공 또는 티웨이) 단독 파트너십 |
| Phase 2 (확장) | 6~18개월 | 4개 광역 연결, MAU 10,000명 | 항공사 2~3개, 지자체 MOU 확대 |
| Phase 3 (글로벌) | 18개월~ | FSC + OTA 연동, 글로벌 인지도 | 대한항공/아시아나 + Agoda/Klook 연동 |

**Phase 1 우선 지역: 속초·양양 (강원 동해권)**
- 이유: 외국인 방문 +124% 급성장 중이나 디지털 멤버십 인프라가 미완성인 '성장 중 미완성' 시장[^3]
- 항공 트리거: 양양국제공항 비엣젯 다낭-양양 노선 탑승률 94.3% 검증[^3]
- 지역화폐: 강원사랑상품권 + 속초 등 시·군 상품권 이중 활용 가능
- 확장 방향: 서울·부산·제주 외에도 청주(충북), 전북(군산·고창), 경남(통영·거제) 등 추가 지역으로 단계적 확장 예정

## 9. 우리 서비스가 아닌 것 (범위 제외)

- 새로운 지역화폐를 발행하는 서비스가 아니다 → 기존 지역화폐를 API로 연결한다
- 항공권 예약 OTA가 아니다 → 탑승 데이터를 연동해 혜택을 연계한다
- 퍼블릭 NFT 거래소가 아니다 → NFT는 관광 보상 수단이며 투자 상품이 아니다
- 국내 내국인 대상 서비스가 아니다 → 핵심 타깃은 방한 외국인이다

## 11. Perplexity 조사 판단 기준

이 서비스에 대해 Perplexity가 자료를 조사하거나 평가할 때, 아래 기준으로 관련성을 판단하라.

**높은 관련성 (우선 조사)**
- Hyperledger Fabric + Public NFT 하이브리드 아키텍처 사례
- 다자간 지역화폐 정산 구조 (Settlement Layer)
- 항공사 탑승 데이터 → NFT/로열티 자동 연동 사례
- 외국인 관광객 대상 Gasless NFT 발행 (Meta Transaction)
- 한국 지역화폐 외국인 사용 정책 및 제도
- 관광 도메인의 블록체인 실제 운영 사례 (KrisPay, Loyyal, Lufthansa UpTrip 등)

**낮은 관련성 (불필요)**
- 암호화폐 투자·거래소 관련 자료
- 내국인 대상 포인트 적립 서비스
- Fabric만 사용하거나 Public NFT만 사용하는 단일 구조 사례 (하이브리드 아닌 경우)
- 마케팅 자료로만 구성되어 기술·수익 구조가 없는 자료
- Fabric 2.2 이하 구버전 기반 자료
