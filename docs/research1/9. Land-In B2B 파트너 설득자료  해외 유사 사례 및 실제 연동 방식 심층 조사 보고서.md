## 1. Executive Summary

### Land-In 파트너 설득에 가장 유효한 해외 사례 Top 5

|------|--------|------------|
| 1 | **Lufthansa Uptrip** (독일) | 탑승권 스캔 → NFT 카드 → 보상 연동. Polygon 블록체인 활용. 실제 운영 중[^1][^2] |
| 2 | **Singapore Airlines Kris+** (싱가포르) | 마일리지 → QR 결제 전환. 가맹점 650+ 확보. 블록체인 기반 정산[^3][^4] |
| 3 | **AirAsia MOVE + BigPay** (말레이시아) | 항공사 + 선불결제 + 슈퍼앱 통합. 6,300만 승객 DB 기반 온보딩[^5][^6] |
| 4 | **Trip.PASS / WOWPASS** (한국) | 외국인 전용 선불결제 + 교통카드. eKYC(여권+안면인식). BC카드 인프라 활용[^7][^8] |
| 5 | **Welcome Suica** (일본) | 공항 즉시 발급, 교통·쇼핑 통합. 디지털 버전 180일 유효. 가맹점 네트워크 광범위[^9][^10] |

### 가장 강한 B2B 설득 포인트 5개

1. **항공사:** "탑승이 소비를 만든다" — Lufthansa Uptrip은 탑승권 스캔 하나로 여행객의 재방문 앱 사용률을 높이는 구조를 실증했다. Land-In은 여기서 한 걸음 더 나아가 지역 소비 데이터를 항공사에 제공한다.[^1][^11]
2. **지자체:** "서울 집중은 데이터로 해결된다" — Kris+는 650개 가맹점에서 QR 결제 데이터로 소비 패턴을 포착한다. Land-In은 동일한 구조를 지방 지자체에 적용해 KPI 측정 가능한 관광 분산 효과를 만든다.[^3][^12]
3. **결제/핀테크:** "외국인 온보딩 비용을 줄인다" — WOWPASS는 출시 18개월 만에 50만 장을 발급하고, 앱 설치자 120만 명을 확보했다. 기존 결제 인프라(BC카드 등)와 제휴하면 별도 라이선스 없이 가맹점 340만 개에 즉시 접근 가능하다.[^13][^14]
4. **가맹점:** "관광객은 스탬프 챌린지에 반응한다" — Uptrip의 카드 컬렉션 게이미피케이션은 실제 반복 방문을 유도한다. Land-In의 NFT 스탬프 구조는 가맹점별 재방문율을 캠페인 단위로 측정 가능하게 한다.[^15]
5. **항공사 + 지자체 공동:** "지방 노선 관광객은 지출이 13% 높다" — Land-In 자체 분석에서 지방+수도권 연계 방문 외래객 1인 지출($1,723)이 서울 단독($1,530) 대비 13% 높다는 데이터는 항공사와 지자체 모두의 핵심 유인이다.

### "지금 가능한 연동" vs "나중에 가능한 연동" 요약

| 단계 | 연동 방식 | 근거 |
|------|-----------|------|
| **지금 가능** | 탑승권 QR/바코드 스캔(IATA BCBP), 예약번호(PNR 코드) 입력, 쿠폰코드/프로모코드 방식, 화이트라벨 선불카드(BC카드/WOWPASS 제휴) | Uptrip이 이미 검증, Trip.PASS 운영 중[^16][^17][^8] |
| **파트너십 MOU 수준** | 항공사 탑승 완료 이벤트 webhook, 정기 배치 정산, 로열티 ID 매핑, KTX/교통 연계 | AirAsia-BigPay 구조처럼 계열사 관계일 때 가능[^18][^6] |
| **Phase 3 이후** | 항공사 PSS 직접 API 연동, 실시간 PNR 데이터 공유, FSC(대한항공/아시아나) 풀 통합 | 협상비용 과도, 규제(GDPR급) 이슈[^19][^20] |

## 3. 핵심 사례 심층 분석

### ① Lufthansa Uptrip — "탑승권 → NFT → 보상" 구조의 실증

**서비스 개요**

Lufthansa Innovation Hub가 개발하고 현재 Miles&More가 운영 중인 모바일 앱이다. 루프트한자 그룹 8개 항공사(Lufthansa, Swiss, Austrian, Brussels Airlines, Eurowings 등) 탑승객이 탑승권을 스캔하면 4장의 NFT 트레이딩 카드 중 2장을 선택·수집하고, 컬렉션을 완성하면 마일리지·라운지·업그레이드 등 보상을 받는다.[^2][^1]

**왜 Land-In과 유사한가**

Land-In처럼 탑승이라는 물리적 행위를 디지털 보상의 트리거로 사용하고, Polygon 블록체인 NFT를 B2C 보상 레이어로 활용한다. 둘 다 기존 항공 로열티(마일리지) 프로그램을 대체하지 않고 보완한다.[^11]

**실제 운영 구조**

- **사용자 흐름:** 탑승 → 앱에서 탑승권 QR 스캔 or 예약코드(PNR) 입력 → NFT 카드 4장 중 2장 선택 → 카드 보유 → 컬렉션 완성 → 마일 등 보상 수령[^16][^17]
- **보안/검증:** 탑승권 바코드는 IATA BCBP 표준 데이터(이름, 편명, 날짜, 좌석, PNR 등)를 포함한다. 앱이 이를 파싱해 서버에서 검증. 탑승 후 6개월 이내 소급 스캔도 허용.[^21][^22]
- **NFT 기술:** Polygon PoS 블록체인. 가스비 없이 발행. 앱 내 마켓플레이스에서 카드 교환 가능. 외부 크립토 지갑에 전송도 가능.[^2][^15]

**실제 연동 방식**

Lufthansa가 탑승권 데이터를 Uptrip에 **직접 API로 실시간 제공하지 않는다.** 사용자가 직접 탑승권을 스캔하거나 예약코드를 입력한다. 즉 **사용자 자가 검증(self-attestation) + 앱 서버 검증** 방식이다. 항공사 PSS와의 직연동 없이도 구현 가능하다.[^17]

**파트너별 가치**

- 항공사: 탑승 데이터 직제공 없이도 게이미피케이션으로 로열티 강화, 마일 소진 채널 확보
- 사용자: 기존 마일리지와 별도로 NFT 컬렉션 보상

**공개 성과**

400+ 유니크 NFT 카드, 20개 컬렉션 운영 중. 이용자 수 및 거래액은 공개되지 않음.[^23]

**한계 / 리스크**

- 항공사 1개 그룹 내 폐쇄 생태계 (외부 항공사 미지원)
- 사용자 스캔 의존 → 위조 탑승권 리스크 (대량 부정수급 방지 필요)
- NFT 가치가 낮으면 사용자 이탈

**Land-In 적용 시사점**

- **즉시 적용 가능:** 탑승권 BCBP 스캔 + PNR 입력 방식은 항공사와 API 직연동 없이 Phase 1부터 구현 가능[^21]
- **개선점:** Land-In은 Uptrip과 달리 QR 결제 + 지역화폐 연동을 추가해야 하므로 NFT 보상 외 실제 경제적 혜택(지역 소비 크레딧)이 차별점

### ② Singapore Airlines Kris+ — "마일 → 일상 소비" 전환의 실증

**서비스 개요**

2018년 'KrisPay'로 출발해 2020년 'Kris+'로 확장된 싱가포르항공의 라이프스타일 앱이다. KrisFlyer 마일리지를 KrisPay 마일로 전환해 가맹점에서 QR 결제로 사용하거나, 가맹점 결제 시 마일리지를 추가로 적립하는 이중 구조다. 2026년 초 기준 싱가포르 내 수백 개 가맹점 운영 중.[^24][^25][^3]

**왜 Land-In과 유사한가**

항공사 로열티 포인트를 지역 가맹점 소비로 연결하는 구조가 Land-In의 "항공 탑승 → 지역 소비 크레딧" 구조와 정확히 일치한다. 블록체인 기반 정산도 동일하다(KPMG + Microsoft 협력 개발).[^26][^27]

**실제 운영 구조**

- **온보딩:** KrisFlyer 회원 = 자동 Kris+ 회원. 여권 불필요 (기존 항공사 계정 활용)
- **결제 흐름:** 앱 실행 → 'Pay' 버튼 → 가맹점 SGQR 코드 스캔 → 마일 비율 설정 → 결제 완료[^12][^4]
- **가맹점 연동:** 싱가포르 SGQR 표준 QR코드로 통합. 가맹점이 별도 단말기 불필요[^12]
- **정산:** KrisPay 마일 ↔ KrisFlyer 마일 상호 전환 가능 (1:1 비율). 유효기간 6개월[^24]
- **마일 가치:** 1 KrisPay 마일 = 0.67 싱가포르 센트 (≈ KrisFlyer 마일의 약 2/3 가치)[^24]

**실제 연동 방식**

- 항공사 로열티 DB ↔ Kris+ 앱: **직접 통합** (계열사 내부 시스템)
- 가맹점 정산: **SGQR + 앱 서버 처리** (블록체인 백엔드 → 불명확, 가맹점에게는 표준 QR 결제처럼 보임)
- 개인정보: 기존 KrisFlyer 동의 기반으로 추가 동의 최소화

**수익 구조**

- 항공사: 마일 소진 채널 확보 (부채 항목 소진 = 비용 절감)
- 가맹점: 고가치 SQ 고객 유입 프리미엄 + 추가 마케팅 채널
- 수익 배분: 공개되지 않음 (불명확)

**공개 성과**

출시 시점 18개 가맹점 → 2020년 150개 파트너 650개 이상 매장 → 2026년 수백 개 운영 중. 정확한 거래액 미공개.[^25][^3]

**한계 / 리스크**

- 항공사 계열사 내부 통합이므로 외부 플랫폼이 동일 방식 복제 불가
- 마일 환율이 낮아 사용자 만족도 낮다는 비판 (1마일 = 0.67센트)[^24]
- 싱가포르 단일 도시 집중 → Land-In의 다지역 구조와 차이

**Land-In 적용 시사점**

- **적용 가능:** 가맹점 QR 코드 + 앱 결제 구조 그대로 채용 가능
- **피해야 할 점:** 마일/포인트 가치가 현금 대비 낮으면 사용자 이탈. Land-In은 지역화폐 크레딧을 1:1 가치로 유지해야 함

### ③ AirAsia MOVE + BigPay — "항공사 슈퍼앱 + 선불결제" 통합 구조

**서비스 개요**

AirAsia가 항공 예약 앱을 'AirAsia MOVE' 슈퍼앱으로 확장하고, BigPay를 독립 핀테크로 분리하되 기본 결제 수단으로 통합한 구조다. BigPay는 Mastercard 기반 선불카드 + 전자지갑으로, 2024년부터 MOVE 앱의 전용 결제 수단이 됐다.[^18][^28]

**왜 Land-In과 유사한가**

항공사 기반 대규모 사용자 DB(6,300만 명)를 선불결제 플랫폼 온보딩에 즉시 활용한 구조다. Land-In이 항공사 파트너십으로 사용자 획득 비용을 절감하려는 전략과 동일하다.[^5]

**실제 운영 구조**

- **BigPay 온보딩:** 앱 설치 → ID 인증 → 선불카드 발급 (Mastercard 네트워크). 글로벌 3,590만 Mastercard 가맹점 이용 가능[^29]
- **AirAsia Xchange:** 파트너 포인트를 BIG Points로 실시간 전환. 세계 최초 항공 포인트 교환 플랫폼[^30]
- **결제 통합:** BigPay 잔액으로 MOVE 앱 내 항공권·호텔 결제 + 외부 가맹점 사용[^18]
- **GetYourGuide 제휴:** 2026년 3월, 관광 액티비티 예약을 MOVE 앱에 통합 (B2B OTA 연동)[^31]

**실제 연동 방식**

- 항공사 ↔ 결제사: **계열사 직통합** (AirAsia 지주회사 Capital A 산하)
- 외부 포인트 전환: **AirAsia Xchange API** (파트너 DB 쿼리 → BIG Points 발행)
- 가맹점 결제: **Mastercard 글로벌 네트워크** (별도 POS 없음)
- 정산 주체: Bank Negara Malaysia 인가 e-money 사업자인 BigPay[^29]

**수익 구조**

- BigPay: FX 환전 수수료, 카드 발급 수수료, 결제 인터체인지
- MOVE 앱: OTA 수수료, 부가서비스 마진

**공개 성과**

AirAsia 63만 DB 활용, BigPay 시리즈 투자 다수 유치. 정확한 MAU, 거래액 비공개.

**한계 / 리스크**

- 계열사 간 통합이어서 독립 플랫폼이 동일 속도로 복제 불가
- AirAsia MOVE는 인천·서울 노선 중심 → 지방 연결 한계

**Land-In 적용 시사점**

- **적용 가능:** BigPay 모델처럼 기존 결제사(BC카드, WOWPASS, KTX)와 제휴형 구조로 시작. 독자 라이선스 없이 지역화폐 인프라 탑승 가능
- **피해야 할 점:** 계열사 통합을 흉내 내려 하면 안 됨. Land-In은 항공사와 자본 관계 없으므로 MOU → 공동 마케팅 → 데이터 제공 순서로 협상해야 함

***

### ④ Trip.PASS (LORDSYSTEM + BC카드) — "여권 eKYC + 선불결제" 현지 사례

**서비스 개요**

한국 스타트업 LORDSYSTEM이 개발한 방한 외국인 전용 모바일 여권 기반 금융·관광 플랫폼이다. BC카드와 제휴해 3.43백만 개 BC카드 가맹점에서 사용 가능한 선불카드를 발급하고, QR 결제·교통카드·VAT 환급·택시 호출까지 통합했다.[^7][^8]

**왜 Land-In과 유사한가**

Land-In의 결제 레이어가 Trip.PASS처럼 기존 카드사 인프라(BC카드)를 활용하는 제휴형 구조를 채택할 경우, 이미 검증된 운영 모델이 된다. 여권 + eKYC 온보딩도 동일하다.

**실제 운영 구조**

- **온보딩:** 여권 스캔 + 안면인식(OCR + 블록체인 기반 여권 검증) → 앱 내 선불카드 즉시 발급[^8]
- **결제:** BC카드 가맹점 340만 개 전국 이용 가능 + Payback QR 가맹점 추가
- **교통:** T-money 기반 교통카드 기능 내장 (지하철·버스·택시 사용)
- **배급 채널:** EastarJet 기내 판매 채널로 물리 카드 판매 시작[^8]
- **VAT 환급:** 앱 내 모바일 세금 환급 기능

**실제 연동 방식**

- 여권 인증: 전자여권 + 안면인식 → 블록체인 해시 저장 (중앙 DB 없음)
- 결제 인프라: **BC카드 QR 결제 인프라 직연동** (BC카드가 acquirer 역할)
- 교통카드: **T-money 협약** (LORDSYSTEM이 운영사에 정산)
- 항공사 연동: EastarJet이 기내 판매 채널 제공 (판매 채널 제휴 수준, boarding-level 검증 아님)

**수익 구조**

- BC카드: 가맹점 수수료 수취 → LORDSYSTEM에 수익 배분 (비율 비공개)
- LORDSYSTEM: 환전 스프레드, 플랫폼 수수료, VAT 환급 수수료

**공개 성과**

출시 초기 단계 (2024년 3월 론칭). 구체적 발급 수 및 거래액 미공개.

**Land-In 적용 시사점**

- **적용 가능:** Land-In도 BC카드 또는 동급 카드사와 화이트라벨 결제 제휴 → 가맹점 별도 개척 없이 전국망 즉시 활용. Phase 1에서 가장 현실적인 결제 인프라 모델
- **피해야 할 점:** Trip.PASS처럼 단독 결제 앱으로만 포지셔닝하면 WOWPASS·Trip.PASS와 직접 경쟁. Land-In은 "멀티시티 NFT 관광 여정 통합"을 차별점으로 명확히 포지셔닝해야 함

***

### ⑤ WOWPASS (Orange Square) — 방한 외국인 선불결제 1위의 온보딩 구조

**서비스 개요**

Orange Square가 운영하는 방한 외국인 전용 선불카드다. 무인 환전기(자체 180대+, CU 편의점 내 확장 중)에서 외화 입금 → 원화 잔액 충전 → 전국 카드 가맹점 결제 + T-money 교통카드 기능. 2022년 7월 출시 후 18개월 만에 50만 장 발급, 앱 120만 설치.[^32][^14][^13]

**왜 Land-In과 유사한가**

Land-In의 사용자 온보딩 문제(외국인이 현지 선불결제 수단을 어떻게 획득하는가)를 WOWPASS가 이미 해결했다. JYP 엔터테인먼트 전략투자를 통해 K-POP 팬덤과 연계한 온보딩 채널도 Land-In이 참고할 수 있다.[^14]

**실제 연동 방식**

- 환전: 자체 무인기(WOW Exchange) → 원화 선불잔액 충전
- 결제: T-money 제휴 (교통카드 겸용) + 카드사 제휴 네트워크
- 온보딩: 여권 불필요, 현금 입금만으로 즉시 사용 가능 (낮은 진입 장벽)
- KYC: 1일 충전 한도 200만원으로 규제 준수

**공개 성과**

- 발급 50만 장 (출시 18개월)[^13]
- 앱 설치 120만 명[^14]
- CU 협약으로 무인기 설치 3배 증가[^13]

**Land-In 적용 시사점**

- **즉시 적용 가능:** WOWPASS 또는 동급 사업자와 API 제휴 → Land-In 앱에서 충전 → 지역화폐 QR 결제로 라우팅
- **차별화:** WOWPASS는 단순 결제 수단. Land-In은 NFT 스탬프 + 멀티시티 혜택 + 항공 연동을 얹어 "관광 플랫폼"으로 차별화

## 6. Land-In 적용 전략 제안

### Phase 1: 가장 현실적인 PoC 구조 (0~6개월)

**추천 파트너 유형:** 소형 LCC 항공사(파라타항공/티웨이) + 강원도 + WOWPASS 또는 BC카드

**추천 연동 방식:**
- **항공 연동:** 탑승권 BCBP 스캔 + PNR 입력 방식 (항공사 API 없이 구현)[^16][^21]
- **결제:** WOWPASS 또는 BC카드 화이트라벨 선불 구조 활용 (독자 라이선스 불필요)[^7][^13]
- **NFT:** Polygon 가스리스 발행 (Uptrip 검증 완료)[^2]
- **보상 트리거:** 탑승권 스캔 → NFT 발행 → 강원 지역 QR 결제 시 지역화폐 크레딧

**왜 현실적인가:** Uptrip이 수백만 탑승객에게 BCBP 스캔 방식을 검증했다. 항공사와 데이터 계약 없이도 구현 가능하다. Trip.PASS가 여권 eKYC를 이미 구현했으므로 온보딩 참고 모델이 있다.[^17][^8]

**절대 과장하면 안 되는 것:** "항공사가 탑승 데이터를 실시간 제공한다" — Phase 1에서 불가능.

***

### Phase 2: 확장 구조 (6~18개월)

**추천 파트너 유형:** 항공사 2~3개 + 4개 광역 지자체 MOU + KTO/Trip.PASS 백본

**추천 연동 방식:**
- **항공 연동:** 공동 마케팅 쿠폰코드 방식 추가 (KTO-PayPay 모델 참고)[^33]
- **결제:** Trip.PASS 제휴 또는 독자 선불 서비스 런칭 (e-money 라이선스 취득)
- **로열티 연동:** 항공사 마일리지 ↔ Land-In 크레딧 교환 (AirAsia Xchange 모델)[^30]
- **지자체 데이터:** 분기별 배치 정산 + 관광 데이터 공동 분석 리포트 제공

**왜 현실적인가:** Kris+가 18개 파트너에서 시작해 650개 이상으로 확장한 경로를 그대로 따를 수 있다. 로열티 ID 매핑은 항공사와 MOU 수준에서 합의 가능하다.[^6][^3]

**절대 과장하면 안 되는 것:** "모든 지자체 지역화폐 API가 표준화됐다" — 각 지역화폐 운영사와 개별 협상 필요.

***

### Phase 3: 장기 이상 구조 (18개월 이후)

**추천 파트너 유형:** 대한항공/아시아나 FSC + Agoda/Klook OTA 연동 + 글로벌 카드사

**추천 연동 방식:**
- **항공 연동:** 제한적 webhook/이벤트 기반 연동 (체크인 완료 이벤트). 실시간 PNR은 현실적으로 불가[^20]
- **OTA 연동:** Klook Open API 방식으로 Land-In 패스를 Klook에 배급[^36]
- **정산 고도화:** Hyperledger Fabric 다자 정산 → 분기 → 월 → 실시간 배치 순으로 고도화

**왜 Phase 3인가:** FSC 항공사는 의사결정이 느리고 데이터 개방에 보수적이다. OTA 연동은 기술보다 수수료 협상이 더 오래 걸린다.

**블록체인이 꼭 필요 없는 부분 (냉정하게):** 가맹점 QR 결제, 교통카드 연동, 관광지 입장권 → 기존 결제사 인프라로 충분. Hyperledger Fabric은 다자 정산 투명성이 필요한 지역화폐 간 교환 레이어에만 집중해야 한다.

## 7. 결론

### Land-In의 해외 비교 우위는 무엇인가?

해외 사례들은 모두 단일 도시(Kris+ → 싱가포르) 또는 단일 항공사 그룹(Uptrip → LH그룹) 내 폐쇄 생태계다. **Land-In이 해결하는 문제 — 한 명의 외국인 관광객이 여러 지방 도시를 넘나들 때 발생하는 결제·보상 파편화 — 는 세계 어디에도 동일한 통합 솔루션이 없다.** Welcome Suica처럼 광범위한 교통망을 커버하는 사례는 있지만, 항공 탑승 + 지역 소비 + NFT 보상을 한 앱으로 묶은 사례는 확인되지 않는다. 이 공백 자체가 비교 우위다.[^9][^1][^3]

### 파트너 설득자료에서 가장 강하게 밀어야 할 한 줄은?

> **"Uptrip은 탑승권 스캔 하나로 루프트한자 로열티를 강화했고, Kris+는 마일을 QR 결제로 연결해 650개 가맹점을 확보했다. Land-In은 그 두 가지를 동시에, 그리고 서울이 아닌 지방에서 처음 해결한다."**

이 문장은 항공사에게 로열티 차별화 근거로, 지자체에게 데이터 기반 관광 분산 근거로, 가맹점에게 신규 고객 유입 근거로 동시에 작동한다.

### 실제 구현 관점에서 가장 먼저 검증해야 할 것은?

**IATA BCBP 스캔 → NFT 발행 → 지역 QR 결제의 단일 플로우를 파라타항공 양양 노선 실승객 100명으로 PoC 완료하는 것이다.** 이 플로우가 작동하는 순간, 나머지 파트너(지자체, 결제사, 추가 항공사)에 대한 협상은 "이미 작동하는 시스템"을 보여주는 데이터 기반 설득으로 전환된다.
